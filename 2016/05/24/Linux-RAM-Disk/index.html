Linux RAM Disk

发表于 2016-05-24   |  

0x00 关于 Linux 相关内存文件系统的类型

  • 关于tmpfs 和 ramfs
    Linux 中有两个基于RAM的文件系统类型，tmpfs和ramfs，ramfs是较老的文件系统类型，tmpfs在大多数情况下是可以取代ramfs的。

  • ramfs
    ramfs 是 Linux 下一种基于 RAM 做存储的文件系统，由于是将 RAM 作为最后一层的存储，所以 ramfs 中不会使用 swap ，但是ramfs的最大缺点就是会吃光系统所有的内存，虽然挂载 ramfs 时候指定了大小，但是但是你可以写入比指定大小更大的文件。

  • tmpfs
    tmpfs 是将所有的文件都保存在虚拟内存中，tmpfs 是 ramfs 的衍生品，tmpfs 使用了虚拟内存机制，所以会进行 swap ，但是相对于 ramfs 的好处是写入的文件，不能超过挂载时所指定的大小。

  • 举例说明
    在 Linux 下使用 free 命令可以获取设备内存相关信息

    root@caydynOS:~# free -h
     total       used       free     shared    buffers     cached
Mem:          2.0G       1.7G       225M         0B       156K        16M
-/+ buffers/cache:       1.7G       242M
Swap:         130M         0B       130M

在 /mnt 下创建挂载点

    mkdir /mnt/ramdisk


+ 使用 ramfs 作为 RAM Disk

        mount -t ramfs -o size=512m tmpfs /mnt/ramdisk

+ 使用 tmpfs 作为 RAM Disk

        mount -t tmpfs -o size=512m tmpfs /mnt/ramdisk
