关于WANem简单实用

发表于 2016-06-23   |  

0x00 关于 WANem

  • WANem简介

    • WANem 全称为 WAN emulator，输一个基于开源的广域网仿真模拟器，其也是基于Linux Traffic Control的，组要提供在局域网环境内模拟广域网的带宽限制、网络时延、网络丢包、时延抖动等网络特性，高级功能有：随机复位连接、按概率重复报文、按概率乱序报文、按概率瞬时断开网络、按协议随机断开网络，可针对不同协议、主机、网络环境，甚至是不同应用程序进行模拟

  • WANem使用优点

    • 完全开源免费
    • 可在单网卡设备上实现所有功能
    • 支持外接设备启动，可无需安装即可正常使用，也可用于虚拟机
    • 可配置多条规则，相互无影响，方便于集中式测试环境
    • 支持WEB图形界面管理，方便设置

  • 下载地址

    • http://wanem.sourceforge.net/（如果无法访问请科学上网）

  • 界面截图

0x01 安装 WANem

  • 下载 WANem 镜像后刻录到U盘中（可使用软碟通等软件刻录）
  • 将U盘插入到可运行设备后，在启动时选择BIOS启动项为U Disk启动
  • 选择U Disk启动后会看到如下界面

  • 当出现 Do you want to configure all interfaces via DHCP（y/n） 时可选择y，有网线连接的网口通过DHCP获取IP地址，选择n则手动配置静态IP地址
  • 最进入到WANem界面

0x02 配置 WANem

  • 进入到 WANem 终端此时控制台的默认提示符为 WANemControl@PERC，输入 help 可以获取帮助信息
  • 输入 exit2shell 可以进入到 Linux 的 Shell 中，通过 ifconfig 命令可以查看当前的设备网口IP地址
  • 可通过 WEB 管理界面配置 WANem，打开浏览器（WANem运行主机 或者 局域网内任意电脑），输入地址 “http://<该机器的IP地址>/WANem”，即可登录控制界面
  • WEB 界面功能导航介绍
    • About 关于，该版本WANem简介
    • WANalyer 广域网学习工具，通过Ping目标主机测试获取丢包、时延、抖动信息
    • Basic Mode 基础模式，只能修改对应网口的带宽、时延参数
    • Advanced Mode 高级模式，可设定更复杂的网络规则，可基于不同接口、主机、网段
      • Packet Limit：用来设置包的队列大小，默认为1000，当队列超过1000时，超过的包将被丢弃
      • Symmetrical network：对称网络，选中为Yes的时候，应用网络的规则将在数据包来回的方向上都生效
      • Choose BW：从快速列表选择带宽
      • Specify BW：自定义带宽
      • Delay：延时，这里设置的是单向的延时。Jitter，抖动；Correlation，相关性，用来设置这个包的延迟时间与上一个包的时间的相关度
      • Loss：丢包率；Correlation，相关性，以一定的概率发生突发的大量的丢包，但平均丢包率不会超过Loss定义的值
      • Duplication：重包率，以一定概率生成某个包的多份拷贝，并按随机时间到达目标端
      • Correlation，相关性，类似Loss的
      • Packet reordering：包重排序率，按概率将包的顺序打乱，gap用来确定包重排序的个数，不设置更接近真实的环境
      • Corruption：错包率，按概率产生噪音，即格式错误的包
      • Idle timer Disconnect：空闲断开定时器，当网络空闲时，按设定的时间对Type定义的协议断开连接
      • Random Disconnect：按Type随机断开，MTTF，平均失效前时间，用Low和High定义两个极值，MTTF，平均恢复前时间
      • IP source address：匹配的源IP地址
      • IP source subnet：源IP地址的子网
      • IP dest address：匹配的目标IP地址
      • IP dest subnet：匹配的目标IP地址的子网
      • Add a rule set：添加一条规则
      • Apply settings：应用设置
    • Save/Restore 备份/恢复数据
    • Help 帮助
  • WANem 在网络拓扑中的位置

0x03 连接到 WANem

  • WANem 为单网口设备
    • Windows 作为客户端
      • 首先删除 Windows 的默认网关：route delete 0.0.0.0
      • 然后增加一条指向 WANem 设备的默认路由：route add 0.0.0.0 mask 0.0.0.0
      • 如若没有生效则，先执行 route -F 后再重复上两条操作
      • 设置时延后，使用Ping可检测效果
    • Linux 作为客户端
      • 首先删除 Windows 的默认网关：route delete 0.0.0.0
      • 然后增加一条指向 WANem 设备的默认路由：route add default gw
      • 设置时延后，使用Ping可检测效果
