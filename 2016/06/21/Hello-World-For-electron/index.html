0x00 关于 electron

  • 什么是 electron

    electron 简单来说就是一个使用 html、css、javascript 开发桌面应用的工具, 最初叫做atom-shell, 后来改名为了electron, 由 Atom 编辑器团队开发构建, 目的是让前端开发的工程师更方便的构建桌面本地应用。

  • electron 制作实例


    更多请点击：http://electron.atom.io/apps/

0x01 搭建 electron 生产环境

文中所用系统环境为 OS X EI Capitan 10.11.5

  • 安装 node.js
    • 在 Node.js 官网下载 OS X 系统PKG安装包，下载地址：https://nodejs.org/en/download/
    • 或者通过 brew 安装
brew install node

  • 安装 electron

    sudo npm install electron-prebuilt -g

如果安装较慢可以使用 TaoBao 的 npm 镜像

sudo npm install -g cnpm --registry=https://registry.npm.taobao.org
sudo cnpm install electron-prebuilt -g

0x02 electron 原理概述

简单来说，Electron为用纯JavaScript创建桌面应用提供了运行时。原理是，Electron调用你在package.json中定义的main文件并执行它。main文件（通常被命名为main.js）会创建一个内含渲染完的web页面的应用窗口，并添加与你操作系统的原生GUI（图形界面）交互的功能。当用Electron启动一个应用，会创建一个主进程。这个主进程负责与你系统原生的GUI进行交互并为你的应用创建GUI（在你的应用窗口）。

仅启动主进程并不能给你的应用用户创建应用窗口。窗口是通过main文件里的主进程调用叫BrowserWindow的模块创建的。每个浏览器窗口会运行自己的渲染进程。渲染进程会在窗口中渲染出web页面（引用了CSS，JavaScript，图片等的HTML文件）。web页面是Chromium渲染的，因为各系统下标准是统一的的，所以兼容性很好。

0x03 创建一个 electron hello world 程序

  • 创建你的程序目录

    your-app/
├── package.json
├── main.js
└── index.html

  • 文件含义
    package.json 程序启动脚本，用于指定主进程，并且列举依赖性
    main.js 程序主进程文件
    index.html 程序界面

  • 编辑你的 package.json

    {
      "name"    : "your-app",
      "version" : "1.0",
      "main"    : "main.js"
}

  • 编辑你的 main.js

    const electron = require('electron');
// Module to control application life.
const {app} = electron;
// Module to create native browser window.
const {BrowserWindow} = electron;

// Keep a global reference of the window object, if you don't, the window will
// be closed automatically when the JavaScript object is garbage collected.
let win;

function createWindow() {
  // Create the browser window.
  win = new BrowserWindow({width: 800, height: 600});

  // and load the index.html of the app.
  win.loadURL(`file://${__dirname}/index.html`);

  // Open the DevTools.
  win.webContents.openDevTools();

  // Emitted when the window is closed.
  win.on('closed', () => {
    // Dereference the window object, usually you would store windows
    // in an array if your app supports multi windows, this is the time
    // when you should delete the corresponding element.
    win = null;
  });
}

// This method will be called when Electron has finished
// initialization and is ready to create browser windows.
// Some APIs can only be used after this event occurs.
app.on('ready', createWindow);

// Quit when all windows are closed.
app.on('window-all-closed', () => {
  // On macOS it is common for applications and their menu bar
  // to stay active until the user quits explicitly with Cmd + Q
  if (process.platform !== 'darwin') {
    app.quit();
  }
});

app.on('activate', () => {
  // On macOS it's common to re-create a window in the app when the
  // dock icon is clicked and there are no other windows open.
  if (win === null) {
    createWindow();
  }
});

  • 编辑你的 index.html

    <!DOCTYPE html>
<html>
  <head>
    <meta charset="UTF-8">
    <title>Hello World!</title>
  </head>
  <body>
    <h1>Hello World!</h1>
    We are using node <script>document.write(process.versions.node)</script>,
    Chrome <script>document.write(process.versions.chrome)</script>,
    and Electron <script>document.write(process.versions.electron)</script>.
  </body>
</html>
