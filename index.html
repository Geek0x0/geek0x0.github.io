关于WANem简单实用

0x00 关于 WANem

  • WANem简介

    • WANem 全称为 WAN emulator，输一个基于开源的广域网仿真模拟器，其也是基于Linux Traffic Control的，组要提供在局域网环境内模拟广域网的带宽限制、网络时延、网络丢包、时延抖动等网络特性，高级功能有：随机复位连接、按概率重复报文、按概率乱序报文、按概率瞬时断开网络、按协议随机断开网络，可针对不同协议、主机、网络环境，甚至是不同应用程序进行模拟

  • WANem使用优点

    • 完全开源免费
    • 可在单网卡设备上实现所有功能
    • 支持外接设备启动，可无需安装即可正常使用，也可用于虚拟机
    • 可配置多条规则，相互无影响，方便于集中式测试环境
    • 支持WEB图形界面管理，方便设置

  • 下载地址

    • http://wanem.sourceforge.net/（如果无法访问请科学上网）

Linux RAM Disk

0x00 关于 Linux 相关内存文件系统的类型

  • 关于tmpfs 和 ramfs
    Linux 中有两个基于RAM的文件系统类型，tmpfs和ramfs，ramfs是较老的文件系统类型，tmpfs在大多数情况下是可以取代ramfs的。

  • ramfs
    ramfs 是 Linux 下一种基于 RAM 做存储的文件系统，由于是将 RAM 作为最后一层的存储，所以 ramfs 中不会使用 swap ，但是ramfs的最大缺点就是会吃光系统所有的内存，虽然挂载 ramfs 时候指定了大小，但是但是你可以写入比指定大小更大的文件。

  • tmpfs
    tmpfs 是将所有的文件都保存在虚拟内存中，tmpfs 是 ramfs 的衍生品，tmpfs 使用了虚拟内存机制，所以会进行 swap ，但是相对于 ramfs 的好处是写入的文件，不能超过挂载时所指定的大小。

WLAN相关名词

1. MSDU (MAC服务数据单元)
  • MPDU就是指的经过802.11协议封装过的数据帧。
  • MSDU是从位于一个协议栈的媒体访问控制（MAC）子层上面的逻辑链路控制（LLC）子层接收到的服务数据单元。LLC和MAC子层被统称为该DLL。
  • MSDU是MAC层服务数据单元，表示MAC层载荷；当MAC层协议数据单元(MPDU)被发送到物理层(PHY)时，它便成为了物理层服务数据单元(PSDU)。
2. MPDU (MAC协议数据单元)
  • MSDU经过添加完整性校验MIC、分帧、添加IV、加密、添加MAC头部后，成为MPDU（MAC Protocol Data Unit，MAC协议数据单元）.
3. A-MSDU (聚合MAC服务数据单元)
  • 将若干个A-MSDU Subframe按照802.11协议格式，封装成一个MPDU报文单元
4. A-MPDU (聚合MAC协议数据单元)
  • 为减小开销，802.11n提出的帧汇聚技术，可把两个以上帧组成一个帧传输。
  • 通过将多个MPDU聚合为一个物理层报文，只需要进行一次新到竞争或退避，既可以完成N个MPDU的同时发送，从而减少了发送N－1个MPDU报文所带来的信道资源消耗。
