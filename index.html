Linux Kernel线程

0x00 Kernel API

使用到的头文件

#include <linux/sched.h>
#include <linux/kthread.h>

相关函数解释

  • 创建 Kernel 线程 

    struct task_struct *kthread_create(int (*threadfn)(void *data),void *data,const char *namefmt, ...);

    参数说明：

    • threadfn 需要执行的主要逻辑
    • data 主逻辑传入参数
    • namefmt 线程名称

创建的线程返回struct task_struct指针，不会立即运行

  • 创建并且启动 Kernel 线程 

    struct task_struct *kthread_run(int (*threadfn)(void *data),void *data,const char *namefmt, ...);

    参数说明：

    • threadfn 需要执行的主要逻辑
    • data 主逻辑传入参数
    • namefmt 线程名称

创建的线程返回struct task_struct指针，会立即运行

  • 启动创建的 Kernel 线程

    int wake_up_process(struct task_struct *tsk);

    参数说明：

    • thread 通过kthread_create创建线程后返回的结构体

  • 线程停止

    int kthread_stop(struct task_struct *thread);

    参数说明：

    • thread 通过kthread_create或者kthread_run创建线程后返回的结构体

  • 线程优先级调整

    int sched_setscheduler(struct task_struct *p, int policy, struct sched_param *param);

    参数说明：

    • p 通过kthread_create创建线程后返回的结构体
    • policy 调度策略类型
参数 含义
SCHED_OTHER 分时调度策略
SCHED_FIFO 实时调度策略，先到先服务
SCHED_RR 实时调度策略，时间片轮转 
    **关于 SHCED_RR 和 SCHED_FIFO 区别**  
    - 当采用SHCED_RR策略的进程的时间片用完，系统将重新分配时间片,并置于就绪队列尾
    - SCHED_FIFO一旦占用cpu则一直运行,一直运行直到有更高优先级任务到达或自己放弃


- **param**  调度优先级

        struct sched_param {
            int sched_priority;
        };
        sched_priority //对应区间 ［1， 99］

0x01 使用实例

  • 实例说明： 

    创建的内核线程每隔一秒做一次打印

struct task_struct *test_task;  

int test_func(void *arg)
{
    uint64_t cur = 0;
    uint64_t last = 0;

    while(!kthread_should_stop())
    {
        cur = get_cycle();
        if (cur - last > 2410000000)
        {
            printk(KERN_INFO "this time: %llu\n", cur);
            last = cur;
        }
    }
    return 0;
}

void run_thread(void)
{
    struct sched_param param = {.sched_priority = 1};

    test_task = kthread_create(&test_func, NULL, "MY_test");
    kthread_bind(test_task, 0);
    sched_setscheduler(test_task, SCHED_FIFO, &param);
    wake_up_process(test_task);

    printk(KERN_INFO"OK, Kernel Thread : %s, Running\n",                 test_task->comm);
}

static int __init test_init(void)
{
    run_thread();
    return 0;
}

static void __exit test_exit(void)
{
    kthread_stop(test_task);
}

module_init(test_init);
module_exit(test_exit);

在Github上使用Hexo搭建个人博客（MAC环境）

0x00 开始准备

  • github

    因为需要用Github充当免费的静态服务器，所以这个是必须具备的

    • 注册 github
      注册链接：Join Github
    • 创建 respository
      这里需要注意 respository 的名字为 [github用户名].github.io
    • 下载使用 github desktop
      下载链接：Github Desktop

  • Node.js

    下载链接: Node.js Download

  • Hexo

    安装Hexo CLI

    npm install hexo-cli -g

0x01 创建博客

  • Hexo

    初始化目录结构

    npm install hexo --save
hexo init

    执行命令后，生成如下目录结构

    .
├── _config.yml
├── package.json
├── scaffolds
├── scripts
├── source
|      ├── _drafts
|      └── _posts
└── themes

    安装Hexo插件： 

    npm install hexo-generator-index --save
npm install hexo-generator-archive --save  
npm install hexo-generator-category --save  
npm install hexo-generator-tag --save  
npm install hexo-server --save  
npm install hexo-deployer-git --save  
npm install hexo-deployer-heroku --save  
npm install hexo-deployer-rsync --save  
npm install hexo-deployer-openshift --save  
npm install hexo-renderer-marked@0.2 --save  
npm install hexo-renderer-stylus@0.2 --save  
npm install hexo-generator-feed@1 --save  
npm install hexo-generator-sitemap@1 --save

    测试结果

    hexo server

    在浏览器中访问 http://localhost:4000

    静态文件

    hexo g

    此时会生成 public 文件夹

  • github

    • 使用 github desktop 软件 clone 之前创建的respository
    • 将之前生成的 public 文件夹中的内容拷贝到 git 文件夹中
    • 提交所有的文件
    • 访问 http://[github用户名].github.io 查看创建好的 Blog

  • Hexo

    hexo n <title> #新建文章
hexo s #本地发布预览
hexo g #生成静态文件
